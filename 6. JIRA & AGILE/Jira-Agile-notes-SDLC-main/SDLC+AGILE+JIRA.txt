What is Jira ?
- Jira ek project management, issue tracking tool aur team collaboration ke liye use hota hai.
- developed by Atlassian.
- Development me Jira ka use hota hai:
                       - Tasks assign karne ke liye
                       - Bug track karne ke liye
                       - Feature development manage karne ke liye
                       - Team ko pata rahe kaun kya kaam kar raha hai
- Example (Software Development Workflow) :
             Maan lo tum Spring Boot project bana rahe ho.
                                             Jira me ek task create hoga:
                                   Task:
                                         Create User Registration API
                                   Isme details hongi:
                                                   Description: User register API banana hai
                                                   Assignee: Meraz Alam
                                                   Priority: High
                                                   Status: To Do → In Progress → Done
                                          Developer (tum) jab kaam start karoge to status In Progress kar doge aur complete hone par Done.

1️⃣ Project Configuration
Meaning:  
      - Project ka initial setup.
      - Manager kya karta hai:
                        - Project create karta hai
                        - Issue types define karta hai (Bug, Task, Story)
                        - Workflow set karta hai
                        - Permissions aur roles set karta hai
    👉 Simple: Project ka environment ready kiya jata hai.


2️⃣ Plan a Version
Meaning:
      - Product ka next release version decide kiya jata hai.
      - Manager kya karta hai:
                     - Version name define karta hai (example: v1.0, v2.0)
                     - Us version me kaun-kaun se features aayenge decide karta hai
                     - Deadline set karta hai
👉 Simple: Next software release ke liye planning.



3️⃣ Plan Sprint
Meaning:
       - Agla sprint (2–3 weeks ka development cycle) plan kiya jata hai.
       - Manager / Scrum Master
                   - Backlog se issues select karta hai
                   - Team members ko tasks assign karta hai
                   - Sprint goal decide karta hai
👉 Simple: Sprint me kya kaam hoga wo decide hota hai.



4️⃣ Run Sprint
Meaning:
      - Team actual development ka kaam karti hai.
      - Users (Developers/Testers):
                              - Coding karte hain
                              - Bugs fix karte hain
                              - Tasks complete karte hain
                              - Daily stand-up meetings hoti hain
👉 Simple: Development ka real work yahan hota hai.



5️⃣ End Sprint
Meaning:
      - Sprint finish hone par review aur retrospective hota hai.
      - Team :
             - Completed tasks check karti hai
             - Demo deti hai
             - Jo incomplete hai wo backlog me wapas chala jata hai
👉 Simple: Sprint ka result evaluate hota hai.



6️⃣ Release Version
Meaning:
       - Jo features sprint me complete hue hain unko software release me convert kiya jata hai.
       - Manager / DevOps:
                     - Build create karte hain
                     - Production me deploy karte hain
👉 Simple: Software ka new version users ko milta hai.



7️⃣ Wrap up Project
Meaning:
      - Project ke remaining tasks close kiye jate hain.
      - Manager :
               - Reports generate karta hai
               - Documentation close karta hai
               - Project archive karta hai
👉 Simple: Project officially complete ho jata hai.




Jira me common issue types
                    Epic → bada feature
                    Story → ek feature ka part
                    Task → normal kaam
                    Bug → error ya problem
       Example:
                    Epic: User Management
                    Story: User Registration
                    Task: Create registration API
                    Bug: Email validation not working


Jira me common boards
        Scrum Board → sprint based development
        Kanban Board → continuous workflow


Developer ke perspective se
Developer Jira me:
             - assigned tickets dekhta hai
             - code likhta hai
             - commit karta hai
             - ticket ka status update karta hai.
