What is Jira ?
- Jira ek project management, issue tracking tool aur team collaboration ke liye use hota hai.
- developed by Atlassian.
- Development me Jira ka use hota hai:
                       - Tasks assign karne ke liye
                       - Bug track karne ke liye
                       - Feature development manage karne ke liye
                       - Team ko pata rahe kaun kya kaam kar raha hai
- Example (Software Development Workflow) :
             Maan lo tum Spring Boot project bana rahe ho.
                                             Jira me ek task create hoga:
                                   Task:
                                         Create User Registration API
                                   Isme details hongi:
                                                   Description: User register API banana hai
                                                   Assignee: Meraz Alam
                                                   Priority: High
                                                   Status: To Do → In Progress → Done
                                          Developer (tum) jab kaam start karoge to status In Progress kar doge aur complete hone par Done.

1️⃣ Project Configuration
Meaning:  
      - Project ka initial setup.
      - Manager kya karta hai:
                        - Project create karta hai
                        - Issue types define karta hai (Bug, Task, Story)
                        - Workflow set karta hai
                        - Permissions aur roles set karta hai
    👉 Simple: Project ka environment ready kiya jata hai.


2️⃣ Plan a Version
Meaning:
      - Product ka next release version decide kiya jata hai.
      - Manager kya karta hai:
                     - Version name define karta hai (example: v1.0, v2.0)
                     - Us version me kaun-kaun se features aayenge decide karta hai
                     - Deadline set karta hai
👉 Simple: Next software release ke liye planning.



3️⃣ Plan Sprint
Meaning:
       - Agla sprint (2–3 weeks ka development cycle) plan kiya jata hai.
       - Manager / Scrum Master
                   - Backlog se issues select karta hai
                   - Team members ko tasks assign karta hai
                   - Sprint goal decide karta hai
👉 Simple: Sprint me kya kaam hoga wo decide hota hai.



4️⃣ Run Sprint
Meaning:
      - Team actual development ka kaam karti hai.
      - Users (Developers/Testers):
                              - Coding karte hain
                              - Bugs fix karte hain
                              - Tasks complete karte hain
                              - Daily stand-up meetings hoti hain
👉 Simple: Development ka real work yahan hota hai.



5️⃣ End Sprint
Meaning:
      - Sprint finish hone par review aur retrospective hota hai.
      - Team :
             - Completed tasks check karti hai
             - Demo deti hai
             - Jo incomplete hai wo backlog me wapas chala jata hai
👉 Simple: Sprint ka result evaluate hota hai.



6️⃣ Release Version
Meaning:
       - Jo features sprint me complete hue hain unko software release me convert kiya jata hai.
       - Manager / DevOps:
                     - Build create karte hain
                     - Production me deploy karte hain
👉 Simple: Software ka new version users ko milta hai.



7️⃣ Wrap up Project
Meaning:
      - Project ke remaining tasks close kiye jate hain.
      - Manager :
               - Reports generate karta hai
               - Documentation close karta hai
               - Project archive karta hai
👉 Simple: Project officially complete ho jata hai.



What is Jira Instance ?
- Jira Instance ka matlab, Jira ko kitne types se use kr skte hai .
- There are two ways to use Jira :
            - Server Instance.
            - Cloud Instance.
👉  Server Instance :
                    - Purches Paid Jira Software version, setup Jira Software on your hosting server. eg- jira.yourhosting.com
                    - need to manage and update time to time.
                    - if you have a big project and a big team then only use server instance.
👉  Cloud Instance:
                    - setup jira software on atlassian.com(jira cloud only)
                    - no server cost.
                    - no security code.
                    - no need to update time to time.
                    - best for learning -> create Account and setup project.

✅ SUMMARY :

             - Issue ka chhota aur clear title hota hai.
2️⃣ Main Idea
             - Task ya bug kis bare me hai, ye quickly batata hai.
3️⃣ Mandatory Field
             - Jira me issue create karte waqt summary fill karna required hota hai.
4️⃣ Quick Identification
             - Team members issue list me summary dekhkar samajh jate hain task kya hai.
5️⃣ Search & Tracking
             - Issues ko search aur track karna easy ho jata hai.

✅TIMELINE :
1️⃣ Time-based view
           - Tasks ko calendar timeline par dikhaya jata hai (week / month basis).
2️⃣ Start & End Dates
           - Har issue ya epic ke liye start date aur due date set ki ja sakti hai.
3️⃣ Planning Tool
           - Project manager ya team lead future work plan kar sakte hain.
4️⃣ Dependency Tracking
           - Ek task dusre task par depend karta hai to timeline me dikhaya ja sakta hai.
5️⃣ Drag & Drop Scheduling

6️⃣ Project Progress Visibility
           - Team easily dekh sakti hai kaunsa task chal raha hai aur kaunsa pending hai.

AGILE METHODLOGY
-----------------

What is Methodlogy in software developement ?
The software development methodology is a process by which developers design, implement and test new computer Software Projects
Eg
How to develop and test Project
How to release the project
How to manage roadmap and Project milestones


What is Agile(फुर्तीला) Methodology ?
The Agile methodology is a way to manage a project by breaking it up into several phases.
Step Example.
Make a plan for 15 days.
Development for plan.
Test project according to the plan.
Make feedback from the customer.
Start Next plan.

Agile Framework :
         - Scrum Framework
         - Kanban Framework
         - Extreme Programming etc


Agile Principle :
12 PRINCIPLES to keep in minds
      - SATISFY THE CUSTOMER
      - WELCOME CHANGING REQUIREMENTS
      - DELIVER WORKING SOFTWARE FREQUENTLY
      - BUSINESS WORKS WITH DEVELOPERS DAILY
      - BUILD PROJECTS AROUND MOTIVATED INDIVIDUALS
      - FACE-TO-FACE CONVERSATION BEST
      - #WORKING SOFTWARE MATTERS
      - CONSTANT PACE
      - HIGH QUALITY CODE IS FASTER
      - MAXIMIZE WORK NOT DONE
      - SELF-ORGANIZING TEAMS DELIVER QUALITY
      - FEEDBACK LOOPS USED TO IMPROVE


Agile Alternatives :
      - Waterfall methodology
      - Rapid Development
      - Lean etc



The Scrum :
      - Scrum is a framework of agile methodology.
      - Helps teams structure and manage their work through a set of values.

Basic Rules in Scrum :
      - Every Sprint is Four Weeks or Less in Duration
      - There are no Breaks Between Sprints
      - The Intention of Every Sprint is "Potentially Shippable" Software
      - Every Sprint includes Sprint Review for stakeholder feedback on the product
Roles in Scrum :
      - Product owner,
      - Scrum master,
      - Development team members
